### Decisions Made During Implementation
- Built umbrella at repo root (orchestra repo IS the musician project root)
- claude-code reference source (src/, tests/, data/) stays as-is — read-only reference
- Mix tasks go in apps/musician_cli/lib/mix/tasks/
- Presets module uses function bodies (not module attributes) for struct literals — compile-time ordering constraint in Elixir
- Schema nested struct defaults set to nil to avoid circular compile-time evaluation
- TUI view modules use tagged tuple trees instead of Ratatouille macros — keeps pure logic layer independently testable
- ex_termbox waf build system needed Python 3 compatibility patches (python → python3 shebang, imp module shim for Python 3.12+)
- OpenAICompat streaming uses Stream.resource stub — full SSE streaming deferred to Phase 2 Week 3 integration
